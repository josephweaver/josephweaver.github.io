# AI Usage Policy for Research (Working Draft v2)

## Purpose
To define responsible, effective use of Large Language Models (LLMs) in research workflows while preserving:
- intellectual ownership
- reproducibility
- technical rigor
- long-term skill development

---

## Core Principle

LLMs are **tools for acceleration and insight**, not substitutes for understanding or authorship.

All publishable work must be:
- **understood**
- **reproducible**
- **defensible without reliance on the LLM**

---

## 1. Exploration Phase (Permissive Use)

LLMs may be used freely for:
- brainstorming ideas and hypotheses
- exploring modeling strategies
- generating prototype code
- suggesting mathematical approaches
- summarizing literature
- designing experiments or pipelines

**Goal:** maximize iteration speed and conceptual exploration

**Guideline:** Treat all outputs as *suggestions*, not verified results

---

## 2. Technical Assistance (Reference-Level Use)

LLMs may be used as an **interactive technical reference**, similar to documentation or community forums.

Examples include:
- “How do I merge dataframes in pandas?”
- “What does this error message mean?”
- “What are possible fixes for this bug?”
- syntax clarification or API usage questions

**Characteristics of this usage:**
- localized and task-specific
- does not introduce opaque system-level logic
- easily verifiable from external sources

**Guideline:**
- acceptable to incorporate directly, provided the researcher understands the result
- must still be tested and validated in context

---

## 3. Development Phase (Guided Use)

LLMs may assist in:
- drafting code structures
- explaining algorithms
- proposing implementations

However, the researcher must:
- understand **every line of code**
- be able to **rewrite the core logic independently**
- verify correctness through testing and reasoning

**Prohibited:**
- blind copy-paste of LLM-generated code into research pipelines
- integrating large opaque components without comprehension

---

## 4. Verification Phase (Strict Ownership)

For any result used in analysis or publication:

### Code
- Re-implement or simplify core components independently
- Validate with:
  - unit tests
  - edge case analysis
  - alternative formulations where feasible

### Mathematics
- Re-derive key results independently
- Confirm assumptions and conditions
- Check limiting cases and consistency

### Models / Results
- Perform robustness checks
- Evaluate sensitivity to inputs and hyperparameters
- Confirm interpretability where applicable

**Standard:** You must be able to explain and defend the result without LLM assistance

---

## 5. Writing and Communication

### Internal Documents
LLMs may be used for:
- drafting
- summarization
- organization

### Publishable Work
- All substantive writing must originate from the researcher
- LLMs may be used **only for:**
  - grammar correction
  - clarity improvements
  - minor stylistic edits

**Prohibited:**
- generating substantive arguments, interpretations, or conclusions via LLM

**Principle:**  
LLM = editor  
Researcher = author

---

## 6. Reproducibility and Transparency

- All code and analysis must be reproducible without LLM interaction
- Workflows should be documented clearly and completely
- LLM usage should be acknowledged at an appropriate level (e.g., brainstorming, prototyping, technical assistance, editing)

---

## 7. Risk Management

Recognized risks of LLM use:
- hallucinated facts or incorrect code
- subtle logical errors
- over-reliance leading to shallow understanding

Mitigation:
- independent verification
- cross-checking with trusted sources
- incremental validation

---

## 8. Productivity Philosophy

We explicitly prioritize:
- **deep understanding over maximum speed**

Target approach:
- leverage LLMs for rapid exploration and assistance
- enforce human ownership through compression, reconstruction, and verification

Expected outcome:
- meaningful productivity gains (approximately 3–5x)
- strong ownership and credibility of results

---

## 9. Guiding Questions (Checklist)

Before including any result in research output:

- Can I explain this without the LLM?
- Can I reproduce this from scratch?
- Do I understand the assumptions?
- Have I tested edge cases or alternatives?
- Would I defend this in a talk, exam, or peer review?

If any answer is “no”, the work is not ready.

---

## 10. Summary

This policy enforces a simple loop:

> LLM output → human understanding → independent reconstruction → verified result

LLMs are used not to replace thinking, but to **accelerate learning and execution while preserving ownership**.
